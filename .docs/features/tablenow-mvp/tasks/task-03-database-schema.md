## Status

## Description

Define the four core domain entities (`User`, `Restaurant`, `TimeSlot`, `Reservation`) and their corresponding EF Core models, Fluent API configurations, and a single `AppDbContext`. Create an initial EF Core migration. The `TimeSlot` entity gets a concurrency token (`RowVersion`) to support optimistic concurrency during reservation creation. This task produces the schema that every backend feature task builds on.

## Technical Details

### Implementation Steps

1. **Add EF Core packages:**
   ```powershell
   dotnet add src/Data/Auth package Microsoft.EntityFrameworkCore
   dotnet add src/Data/Restaurants package Microsoft.EntityFrameworkCore
   dotnet add src/Data/Reservations package Microsoft.EntityFrameworkCore
   dotnet add src/Migrations package Microsoft.EntityFrameworkCore.Design
   dotnet add src/Migrations package Microsoft.EntityFrameworkCore.Sqlite
   dotnet add src/Api package Microsoft.EntityFrameworkCore.Sqlite
   dotnet add src/Api package Microsoft.EntityFrameworkCore.SqlServer
   ```

2. **Create domain entities** (plain C# — no EF attributes).

3. **Create EF models** with EF-specific attributes (e.g., `[Timestamp]` on `RowVersion`).

4. **Create Fluent API configurations** implementing `IEntityTypeConfiguration<TModel>`.

5. **Create `AppDbContext`** in the Migrations project, registering all `DbSet`s and calling `ApplyConfigurationsFromAssembly` for each Data assembly.

6. **Register `AppDbContext` in DI** in `ServiceCollectionExtensions` (Api project):
   ```csharp
   services.AddDbContext<AppDbContext>(options =>
       options.UseSqlite(configuration.GetConnectionString("DefaultConnection")));
   ```

7. **Run migrations** (human step — see action-required.md):
   ```powershell
   dotnet ef migrations add InitialCreate --project server/src/Migrations/TableNow.Migrations.csproj --startup-project server/src/Api/TableNow.Api.csproj
   ```

**EF Configuration for TimeSlot:**
```csharp
// server/src/Data/Restaurants/Configurations/TimeSlotConfiguration.cs
namespace TableNow.Restaurants.Data.Configurations;

public sealed class TimeSlotConfiguration : IEntityTypeConfiguration<TimeSlotModel>
{
    public void Configure(EntityTypeBuilder<TimeSlotModel> builder)
    {
        builder.HasKey(x => x.Id);
        builder.Property(x => x.RowVersion).IsRowVersion();
        builder.Property(x => x.Date).IsRequired();
        builder.Property(x => x.Time).IsRequired();
        builder.HasIndex(x => new { x.RestaurantId, x.Date });
        builder.HasOne(x => x.Restaurant)
               .WithMany()
               .HasForeignKey(x => x.RestaurantId);
    }
}
```

**`AppDbContext`:**
```csharp
// server/src/Migrations/AppDbContext.cs
namespace TableNow.Migrations;

public sealed class AppDbContext(DbContextOptions<AppDbContext> options) : DbContext(options)
{
    public DbSet<UserModel> Users => Set<UserModel>();
    public DbSet<RestaurantModel> Restaurants => Set<RestaurantModel>();
    public DbSet<TimeSlotModel> TimeSlots => Set<TimeSlotModel>();
    public DbSet<ReservationModel> Reservations => Set<ReservationModel>();

    protected override void OnModelCreating(ModelBuilder modelBuilder)
    {
        modelBuilder.ApplyConfigurationsFromAssembly(
            typeof(UserConfiguration).Assembly);      // Auth.Data
        modelBuilder.ApplyConfigurationsFromAssembly(
            typeof(RestaurantConfiguration).Assembly); // Restaurants.Data
        modelBuilder.ApplyConfigurationsFromAssembly(
            typeof(ReservationConfiguration).Assembly); // Reservations.Data
    }
}
```

## Notes

- Domain entities in `Domain/<Context>/` are plain C# — no EF attributes. EF attributes live only on the model classes in `Data/<Context>/Models/`.
- For MVP, the domain entity and EF model shapes are identical — resist the urge to add a mapping layer. Use the EF model directly in data handlers.
- `[Timestamp]` maps to a `rowversion` column in SQL Server and is automatically handled as a concurrency token. For SQLite (dev), EF Core emulates it.
- `DateOnly` and `TimeOnly` require EF Core 6+ — both are available in EF Core 10.
