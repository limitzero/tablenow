# Task 03: Migrations Project & Initial Migration

## Description

Creates the three Migrations projects (one per business context), wires the DbContexts into the API startup for migration discovery, and generates the initial EF migration for each context. After this task, `dotnet ef database update` creates all four tables in the local SQLite database.

## Technical Details

### Implementation Steps

1. Add project references from each Migrations project to its corresponding Data project (if not already done in STORY-001):
   ```powershell
   dotnet add server/src/Migrations/CM.TableNow.Auth.Migrations reference server/src/Data/CM.TableNow.Auth.Data
   ```

2. Create a design-time factory for each DbContext (needed for `dotnet ef` to discover them):

```csharp
// AuthDbContextFactory.cs
using Microsoft.EntityFrameworkCore;
using Microsoft.EntityFrameworkCore.Design;

namespace CM.TableNow.Auth.Migrations;

public class AuthDbContextFactory : IDesignTimeDbContextFactory<AuthDbContext>
{
    public AuthDbContext CreateDbContext(string[] args)
    {
        var options = new DbContextOptionsBuilder<AuthDbContext>()
            .UseSqlite("Data Source=tablenow.db")
            .Options;
        return new AuthDbContext(options);
    }
}
```

3. Register DbContexts in `ServiceCollectionExtensions.cs`:
```csharp
public static IServiceCollection RegisterServices(
    this IServiceCollection services,
    IConfiguration configuration)
{
    var connectionString = configuration.GetConnectionString("DefaultConnection")
        ?? "Data Source=tablenow.db";

    services.AddDbContext<AuthDbContext>(opts => opts.UseSqlite(connectionString));
    services.AddDbContext<RestaurantsDbContext>(opts => opts.UseSqlite(connectionString));
    services.AddDbContext<ReservationsDbContext>(opts => opts.UseSqlite(connectionString));

    return services;
}
```

4. Generate migrations for each context from `server/`:
   ```powershell
   dotnet ef migrations add InitialCreate `
     --project src/Migrations/CM.TableNow.Auth.Migrations `
     --startup-project src/Api `
     --context AuthDbContext

   dotnet ef migrations add InitialCreate `
     --project src/Migrations/CM.TableNow.Restaurants.Migrations `
     --startup-project src/Api `
     --context RestaurantsDbContext

   dotnet ef migrations add InitialCreate `
     --project src/Migrations/CM.TableNow.Reservations.Migrations `
     --startup-project src/Api `
     --context ReservationsDbContext
   ```

5. Apply migrations:
   ```powershell
   dotnet ef database update `
     --project src/Migrations/CM.TableNow.Auth.Migrations `
     --startup-project src/Api

   dotnet ef database update `
     --project src/Migrations/CM.TableNow.Restaurants.Migrations `
     --startup-project src/Api

   dotnet ef database update `
     --project src/Migrations/CM.TableNow.Reservations.Migrations `
     --startup-project src/Api
   ```

### Environment Variables

- `ConnectionStrings__DefaultConnection` — Override with SQL Server connection string for production. Default: `Data Source=tablenow.db` (SQLite for local dev).

## Notes

The design-time factory hardcodes SQLite for local `dotnet ef` CLI usage. The runtime uses the connection string from `appsettings.json` / environment variable. This is intentional — the factory is only used by the EF tooling, not at runtime.
